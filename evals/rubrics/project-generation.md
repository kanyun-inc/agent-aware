# 项目生成质量评估

评估 LLM 生成的测试项目的质量和完整性

## 评估标准

### project_structure (权重: 3)

项目结构是否完整，包含所有必要文件

| 分数 | 描述 |
|------|------|
| 3 | 包含所有必要文件（package.json, vite.config, index.html, src/main.jsx, src/App.jsx），结构清晰 |
| 2 | 缺少部分非关键文件，但项目可以运行 |
| 1 | 缺少多个文件，项目勉强能运行 |
| 0 | 结构严重不完整，项目无法运行 |

### sdk_integration (权重: 3)

agent-aware SDK 是否正确集成

| 分数 | 描述 |
|------|------|
| 3 | SDK 在入口文件顶部正确初始化，配置参数正确 |
| 2 | SDK 初始化位置略有偏差，但功能正常 |
| 1 | SDK 初始化存在问题，部分功能可能受影响 |
| 0 | SDK 未集成或集成错误 |

### bug_diversity (权重: 2)

预埋问题的多样性和真实性

| 分数 | 描述 |
|------|------|
| 2 | 包含所有三种类型的问题（dead_click, rage_click, runtime_error），场景真实 |
| 1 | 问题类型不完整或场景过于简单 |
| 0 | 问题设计不合理或无法触发 |

### code_quality (权重: 2)

生成代码的质量和可读性

| 分数 | 描述 |
|------|------|
| 2 | 代码规范、可读性好、无语法错误 |
| 1 | 代码有小问题但可运行 |
| 0 | 代码有严重问题或无法运行 |
