# Skill 遵循度评估

评估 Agent 是否按照 Skill 文档的步骤和最佳实践执行任务

## 评估标准

### workflow_order (权重: 3)

是否按 Skill 定义的工作流顺序执行（Server → SDK → Query）

| 分数 | 描述 |
|------|------|
| 3 | 完全按照 Skill 定义的顺序执行，没有遗漏步骤 |
| 2 | 顺序基本正确，有小的偏差但不影响功能 |
| 1 | 顺序混乱但最终完成了任务 |
| 0 | 完全忽略工作流，胡乱执行 |

### command_accuracy (权重: 3)

使用的命令是否与 Skill 文档中定义的一致

| 分数 | 描述 |
|------|------|
| 3 | 使用的命令与 Skill 文档完全一致 |
| 2 | 命令基本正确，有参数调整但合理 |
| 1 | 命令部分正确，有明显偏差 |
| 0 | 使用了错误的命令或自创命令 |

### best_practices (权重: 2)

是否遵循 Skill 文档中的 Best Practices 部分

| 分数 | 描述 |
|------|------|
| 2 | 遵循了所有最佳实践建议 |
| 1 | 遵循了部分最佳实践 |
| 0 | 忽略了最佳实践 |

### error_handling (权重: 2)

是否按 Skill 文档处理错误情况（如检查 health 端点）

| 分数 | 描述 |
|------|------|
| 2 | 正确检查并处理了可能的错误情况 |
| 1 | 部分检查了错误情况 |
| 0 | 没有进行任何错误检查 |
