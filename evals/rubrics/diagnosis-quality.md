# 问题诊断能力评估

评估 Agent 检测到问题后的诊断和修复能力

## 评估标准

### issue_identification (权重: 3)

是否正确识别问题类型（rage_click/dead_click/runtime_error）

| 分数 | 描述 |
|------|------|
| 3 | 准确识别问题类型并理解其含义 |
| 2 | 识别了问题但理解不完全准确 |
| 1 | 部分识别问题 |
| 0 | 未能识别问题或识别错误 |

### root_cause_analysis (权重: 4)

是否分析了问题根因而非仅描述现象

| 分数 | 描述 |
|------|------|
| 4 | 深入分析了问题根因，找到了代码层面的原因 |
| 3 | 分析了可能的原因，基本正确 |
| 2 | 提供了一些分析但不够深入 |
| 1 | 只描述了现象，没有分析原因 |
| 0 | 没有进行任何分析 |

### fix_effectiveness (权重: 3)

修复方案是否有效解决了问题

| 分数 | 描述 |
|------|------|
| 3 | 修复方案准确有效，按 Skill 文档建议实施 |
| 2 | 修复方案基本有效 |
| 1 | 修复方案部分有效或引入新问题 |
| 0 | 修复无效或没有提供修复 |
